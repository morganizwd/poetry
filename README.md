# Интеллектуальная система генерации стихов

Учебный проект по генерации русскоязычных стихотворений. В проекте реализованы два подхода:

1. Генерация с помощью цепей Маркова.
2. Генерация с помощью LSTM-модели с простым механизмом attention.

Проект рассчитан на запуск в Google Colab. Внешние LLM, Ollama и другие большие языковые модели здесь не используются.

## Простое описание для клиента

Эта система учится на готовом наборе стихотворений и затем пробует создавать новые строки в похожем стиле.

В проекте есть два варианта генерации:

- Цепи Маркова быстро собирают новые строки на основе вероятных переходов между словами.
- LSTM обучается предсказывать следующее слово по предыдущим словам и за счёт этого строит строки более гибко.

Дополнительно система пытается подбирать рифмы по окончаниям слов. Пользователь может выбрать схему рифмовки, например `AABB`, `ABAB`, `ABBA` или `AAAA`.

После генерации выводятся простые метрики:

- насколько часто удалось подобрать рифму;
- насколько похожи окончания рифмующихся строк;
- сколько уникальных строк получилось;
- средняя длина строки;
- количество гласных;
- количество уникальных слов в стихотворении.

Метрики не являются строгой литературной оценкой качества. Они нужны, чтобы примерно сравнивать результаты разных запусков.

## Запуск генерации через LSTM

Перейдите в папку LSTM-части:

```python
%cd /content/poetry/LSTM_generation_2
```

Запустите скрипт:

```python
!python lstm_generation.py
```

При первом запуске модель начнёт обучение. Это может занять время, особенно если датасет большой.

По умолчанию используется 10 эпох:

```python
generator.load_and_train("poems_clean.txt", epochs=10)
```

Если нужно быстрее проверить, что код работает, можно временно уменьшить число эпох в файле `lstm_generation.py`, например до 3:

```python
generator.load_and_train("poems_clean.txt", epochs=3)
```

В Colab TensorFlow может вывести предупреждения про `cuFFT`, `cuDNN`, `cuBLAS` или `computation placer already registered`. Обычно это не ошибка. Если после строки `Обучение модели` долго нет вывода, модель просто обучается. В актуальной версии прогресс печатается после каждой эпохи:

```text
Epoch 1/10, Loss: ...
Epoch 2/10, Loss: ...
```

После обучения модель сохраняется в файл:

```text
poet_model.keras
```

Также сохраняются служебные файлы:

```text
poet_metadata.json
poet_rhymes.json
```

При следующем запуске код попробует загрузить сохранённую модель, если она подходит к текущей версии кода.

## Что вводить при запуске

Оба скрипта работают примерно одинаково.

Можно оставить начальную строку пустой:

```text
Введите начальную строку:
>
```

Тогда программа начнёт генерацию сама.

Можно задать свою первую строку:

```text
Введите начальную строку:
> Я помню чудное мгновенье
```

После этого программа попробует продолжить стихотворение и подобрать рифмы к нужным строкам.

Доступные схемы рифмовки:

```text
AABB - первая строка рифмуется со второй, третья с четвёртой
ABAB - первая строка рифмуется с третьей, вторая с четвёртой
ABBA - первая строка рифмуется с четвёртой, вторая с третьей
AAAA - все строки пытаются рифмоваться между собой
```

## Возможные ошибки

### Ошибка: `Файл 'poems_clean.txt' не найден`

Причина: датасет не скопирован в папку нужного скрипта.

Решение:

```python
!cp /content/poems_clean.txt /content/poetry/Markov_chain_2/poems_clean.txt
!cp /content/poems_clean.txt /content/poetry/LSTM_generation_2/poems_clean.txt
```

### Ошибка: `No module named markovify`

Причина: не установлена библиотека для цепей Маркова.

Решение:

```python
!pip install -q markovify
```

### Ошибка: `No module named tensorflow`

Причина: в окружении нет TensorFlow.

Решение:

```python
!pip install -q tensorflow
```

### LSTM обучается слишком долго

Это нормально для нейросетевой модели. Для быстрой проверки можно уменьшить количество эпох:

```python
generator.load_and_train("poems_clean.txt", epochs=3)
```

Для финального запуска лучше вернуть больше эпох, например 10.

### Стихотворение не сгенерировалось

Возможные причины:

- слишком маленький датасет;
- мало слов с подходящими рифмами;
- слишком строгая проверка уникальности;
- модель LSTM ещё плохо обучилась.

Обычно помогает:

- использовать больший датасет;
- попробовать другую схему рифмовки;
- оставить начальную строку пустой;
- для LSTM увеличить количество эпох.

## Краткое сравнение подходов

Цепи Маркова:

- работают быстро;
- проще устроены;
- не требуют долгого обучения;
- иногда дают более механические строки.

LSTM:

- обучается дольше;
- требует TensorFlow;
- лучше учитывает последовательность слов;
- результат сильнее зависит от качества и размера датасета.
